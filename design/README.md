# Technical Design Docs (Feature & Service RFC)

All feature/service design docs. Write before any large change/feature implementation.

## Process
- RFC required for large refactor/new feature/cross-cutting concerns.
- Template: Google style (goals, background, proposal, security, alternatives, rollout).
- Filename: `YYYY-MM-key-feature.md` (e.g., `2025-11-global-search-design.md`)
- Review: all platform/integration PO, tech lead, SRE for ops-impact.

See `feature-design-template.md` in this folder.
